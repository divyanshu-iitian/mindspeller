- Flask backend with a `/api/chat` endpoint
- `.env` based API key loading using `python-dotenv`
- Direct Groq API access through HTTP, avoiding newer SDK dependency issues
- Next.js + TypeScript frontend with chat history and loading state
- CORS and environment-based configuration for local and production deployment
- Python 3.8 pinned runtime support through Docker and compatibility checks in CI
